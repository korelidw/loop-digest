From Chat to Coordinated Memory + Multi‑Agent Workflow Overview

Audience: engineering/product colleagues evaluating how to move beyond ephemeral chat toward durable, auditable, multi‑agent work.

Problem

Pure chat is ephemeral: context fades, decisions aren’t traceable, and results aren’t reproducible.

Concept

Add a persistent, human‑readable memory plus role‑based agents and a deterministic data pipeline. The AI becomes a narrator/critic anchored to artifacts, not a free‑text oracle.

Key Components

  1. Deterministic pipeline (source of truth)
    Nightscout fetch → analyses → artifacts (data/*.json, dist/*.svg) → dashboard (site/index‑YYYYMMDD‑HHMM.html)
  2. Structured memory (files, not a black box)
    • memory/runs/YYYY‑MM‑DD.md: daily journal of record used by all agents
    • memory/shared/*: definitions, guardrails, open_questions, literature, experiment_ledger, decisions
    • memory/agents/<role>/scratch.md: proposals/diffs awaiting approval
    • memory/YYYY‑MM‑DD.md: durable end‑of‑day highlights
  3. Role‑based agents (separation of concerns)
    • Analyst → data quality/KPIs, directional hypotheses
    • Researcher → testable hypotheses, falsification tests + evidence
    • Code Agent → UI/content proposals, safe diffs (await approval)
    • Logging Audit → data fidelity + clean‑only contrasts
    • Release → promote durable outcomes; keep a single prioritized backlog
  4. Guardrails: evidence‑backed, directional recommendations; uncertainty and safety notes; no automatic therapy changes.

Why This Beats Chat

  • Traceability: every claim points to artifacts (data JSON, site build, commit SHA, literature IDs)
  • Repeatability: jobs + memory yield consistent, auditable outputs
  • Collaboration: specialized agents share structured context, not just token history
  • Safety: guardrails constrain outputs to decision support

Where to Look

  • /data → analysis JSON; /dist → SVGs; /site → dashboards
  • /memory/runs → today’s journal; /memory/shared → durable context; /memory/agents/*/scratch.md → per‑role backlogs

How It Coordinated in Our Run

  • Advanced Evidence Mode toggled in shared/definitions.md; agents started including explicit n, medians/IQRs, %ineffective@2h.
  • Code Agent staged safe UI tweaks (Advanced badge, KPI legend, Experiment Peek empty‑state, “What to check today” chips); we approved and published.
  • Researcher reconciled literature vs data (midday sensitivity) and prioritized stratifying corrections by constraint state before ISF discussion.
  • Release consolidated outcomes, updated open questions, and linked dashboard builds + literature IDs.

Next Steps (Generalizable)

  • Formalize exclusive KPI bins (no overlaps) and add tests.
  • Enrich correction context with per‑event constraint state; surface gating chips in heatmap rows.
  • Add a lightweight PR flow for Code Agent diffs prior to publish.
